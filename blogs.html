What is robots.txt and how have you configure it for your site?

  A robots.txt file tells search engine crawlers(also known as robots,spiders) which pages or files the crawler can or can't request from your site. This is used mainly to avoid overloading site with requests. Ideally, a robots.txt file is placed in the top-level directory of website so that robots can access its instructions right away.It lists the URLs that the webmaster doesn’t want Google or any search engine to index and prevents them from visiting and tracking the selected pages. When a bot finds a website on the Internet, the first thing it does is check the robots.txt file in order to learn what it is allowed to explore and what it has to ignore during the crawl.

In my webpage I have created a robots.txt file and I have stopped all search engines to index the contact page.

User-Agent:*

Disallow : /contact.html

What is humans.txt and how have you configure it for your site?

  It's an initiative for knowing the people behind a website. It's a TXT file that contains information about the different people who have contributed to building the website.

Where is it located? In the site root. Just next to the robots.txt file. If possible, you can also add an author tag to the head of the site.

In my site I created a human.txt file.

Placed a reference to the file in the < head> of the site: < link rel="author" href="humans.txt" />
And in the human.txt file I gave some information:

What is Open Graph and how do you make use of it on your site?

  Open Graph is a technology first introduced by Facebook in 2010 that allows integration between Facebook and its user data and a website. By integrating Open Graph meta tags into your page's content, you can identify which elements of your page you want to show when someone share's your page. To turn my webpage into a graph object, I used open graphs protocol meta tags such as:

< meta property="og:type" content="blog"/>
< meta property="og:title" content="Lena's blog"/>
< meta property="og:url" content="https://1dv525.github.io/eh223ti-examination-1/blogs.html"/>
< meta property="og:image" content="https://ecobnb.it/blog/app/uploads/sites/2/2018/07/Axels_n%C3%B6jesf%C3%A4lt._V%C3%A4xj%C3%B6.jpg"/>
< meta property="og:description" content="A view on my new life in Sweden"/>

which I put on the head of my html pages.
Facebook offers an Open Graph debugger that allows anyone to test their page and determine how it will appear when shared on Facebook. I opened Facebook for Developers then Sharing Debugger where I could see link preview. In the end I opened Open Graph Object Debugger and shared it on Facebook.

Why is it important to seperate the layout and design (css) from content (html)?

  If we use Inline stylesheets we specify for a specific element and need to copy for the others. Also if we use Embedded stylesheets every page should have the same style in the head and they will download the same style everytime they are opened. If we seperate the design from the content in seperate files, browser downloads mystyle.css and when the user goes to next page the browser will load the next html file and will find that the file .css it needs, is cashed in the browser and the site will load faster.
